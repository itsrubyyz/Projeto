# Meu Projeto

## Fluxo de Aprovação (Pull Requests)

Todos os Pull Requests (PRs) devem seguir este fluxo:

1. Crie uma branch para sua funcionalidade ou correção.
2. Abra um Pull Request apontando para a branch `develop`.
3. Solicite revisão de pelo menos um membro da equipe.
4. O revisor deve comentar no PR, confirmando os critérios:
   - Funcionalidade implementada corretamente
   - Código sem erros aparentes
   - Não compromete outras partes do sistema
   - Organização e legibilidade do código
5. Após aprovação, faça o merge.
